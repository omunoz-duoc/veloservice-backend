# CI/CD — GitHub Actions → Artifact Registry → Cloud Run

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Push to `main` triggers: test → build Docker image → push to Google Artifact Registry → deploy to Cloud Run.

**Architecture:** GitHub Actions workflow runs Maven tests with H2 profile, builds a multi-stage Docker image (Java 21 / Spring Boot fat JAR), pushes to GAR via Workload Identity Federation (no long-lived keys), and deploys to Cloud Run with environment variables injected from GitHub Secrets.

**Tech Stack:** GitHub Actions, Docker (multi-stage), Google Artifact Registry, Google Cloud Run, Workload Identity Federation, Maven 3.9 / Java 21, Spring Boot 3.3

## Pre-requisites (Manual GCP Setup — do once in terminal)

These steps run **outside** this repo. Complete them before executing tasks below.

### A. GCP Project & APIs

```bash
# Set your project
export PROJECT_ID=your-gcp-project-id
export REGION=us-central1
export REPO_NAME=veloservice
export SERVICE_NAME=veloservice-backend
export GITHUB_ORG=your-github-username-or-org
export GITHUB_REPO=veloservice-backend

gcloud config set project $PROJECT_ID

# Enable required APIs
gcloud services enable \
  artifactregistry.googleapis.com \
  run.googleapis.com \
  iam.googleapis.com \
  iamcredentials.googleapis.com \
  sts.googleapis.com
```

### B. Artifact Registry repository

```bash
gcloud artifacts repositories create $REPO_NAME \
  --repository-format=docker \
  --location=$REGION \
  --description="Veloservice backend images"
```

### C. Service Account + Workload Identity Federation

```bash
# Service account that GitHub Actions will impersonate
gcloud iam service-accounts create github-actions-sa \
  --display-name="GitHub Actions SA"

SA_EMAIL="github-actions-sa@${PROJECT_ID}.iam.gserviceaccount.com"

# Grant roles
gcloud projects add-iam-policy-binding $PROJECT_ID \
  --member="serviceAccount:${SA_EMAIL}" \
  --role="roles/artifactregistry.writer"

gcloud projects add-iam-policy-binding $PROJECT_ID \
  --member="serviceAccount:${SA_EMAIL}" \
  --role="roles/run.admin"

gcloud projects add-iam-policy-binding $PROJECT_ID \
  --member="serviceAccount:${SA_EMAIL}" \
  --role="roles/iam.serviceAccountUser"

# Workload Identity Pool
gcloud iam workload-identity-pools create github-pool \
  --location=global \
  --display-name="GitHub Actions Pool"

POOL_ID=$(gcloud iam workload-identity-pools describe github-pool \
  --location=global --format="value(name)")

# OIDC Provider for GitHub
gcloud iam workload-identity-pools providers create-oidc github-provider \
  --workload-identity-pool=github-pool \
  --location=global \
  --issuer-uri="https://token.actions.githubusercontent.com" \
  --attribute-mapping="google.subject=assertion.sub,attribute.repository=assertion.repository,attribute.actor=assertion.actor,attribute.ref=assertion.ref" \
  --attribute-condition="assertion.repository=='${GITHUB_ORG}/${GITHUB_REPO}'"

PROVIDER_NAME=$(gcloud iam workload-identity-pools providers describe github-provider \
  --workload-identity-pool=github-pool \
  --location=global \
  --format="value(name)")

# Bind SA to pool
gcloud iam service-accounts add-iam-policy-binding $SA_EMAIL \
  --role="roles/iam.workloadIdentityUser" \
  --member="principalSet://iam.googleapis.com/${POOL_ID}/attribute.repository/${GITHUB_ORG}/${GITHUB_REPO}"

# Print values needed for GitHub Secrets
echo "WIF_PROVIDER: ${PROVIDER_NAME}"
echo "WIF_SERVICE_ACCOUNT: ${SA_EMAIL}"
echo "GAR_LOCATION: ${REGION}-docker.pkg.dev/${PROJECT_ID}/${REPO_NAME}"
```

```dockerfile
# Stage 1 — build
FROM maven:3.9.6-eclipse-temurin-21-alpine AS build
WORKDIR /app
COPY pom.xml .
# Download deps first for layer caching
RUN mvn dependency:go-offline -B -q
COPY src ./src
# Skip tests here; CI already ran them
RUN mvn package -DskipTests -B -q

# Stage 2 — runtime
FROM eclipse-temurin:21-jre-alpine
WORKDIR /app
# Non-root user for security
RUN addgroup -S spring && adduser -S spring -G spring
USER spring:spring
COPY --from=build /app/target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

## Task 3: GitHub Actions CI/CD workflow

**Files:**
- Create: `.github/workflows/ci-cd.yml`

- [ ] **Step 1: Create `.github/workflows/` directory and workflow file**

```bash
mkdir -p .github/workflows
```

- [ ] **Step 2: Write `.github/workflows/ci-cd.yml`**

```yaml
name: CI/CD — Build, Push & Deploy

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

env:
  GAR_LOCATION: ${{ secrets.GCP_REGION }}-docker.pkg.dev
  IMAGE: ${{ secrets.GCP_REGION }}-docker.pkg.dev/${{ secrets.GCP_PROJECT_ID }}/${{ secrets.GAR_REPOSITORY }}/veloservice-backend

jobs:
  # ── 1. Test ──────────────────────────────────────────────────────────────
  test:
    name: Run Tests
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up JDK 21
        uses: actions/setup-java@v4
        with:
          java-version: '21'
          distribution: 'temurin'
          cache: maven

      - name: Run tests (H2 profile)
        run: mvn test -B -Dspring.profiles.active=h2

      - name: Upload test results
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: test-results
          path: target/surefire-reports/

  # ── 2. Build & Push ───────────────────────────────────────────────────────
  build-and-push:
    name: Build & Push Image
    runs-on: ubuntu-latest
    needs: test
    # Only push on main; PRs only run tests
    if: github.ref == 'refs/heads/main' && github.event_name == 'push'
    permissions:
      contents: read
      id-token: write   # Required for Workload Identity Federation

    outputs:
      image-tag: ${{ steps.meta.outputs.version }}
      full-image: ${{ env.IMAGE }}:${{ steps.meta.outputs.version }}

    steps:
      - uses: actions/checkout@v4

      - name: Authenticate to Google Cloud (WIF)
        id: auth
        uses: google-github-actions/auth@v2
        with:
          workload_identity_provider: ${{ secrets.WIF_PROVIDER }}
          service_account: ${{ secrets.WIF_SERVICE_ACCOUNT }}

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Configure Docker for GAR
        run: gcloud auth configure-docker ${{ env.GAR_LOCATION }} --quiet

      - name: Extract Docker metadata (tag = short SHA)
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ${{ env.IMAGE }}
          tags: |
            type=sha,prefix=,format=short
            type=raw,value=latest

      - name: Build and push image
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

  # ── 3. Deploy to Cloud Run ────────────────────────────────────────────────
  deploy:
    name: Deploy to Cloud Run
    runs-on: ubuntu-latest
    needs: build-and-push
    permissions:
      contents: read
      id-token: write

    steps:
      - name: Authenticate to Google Cloud (WIF)
        uses: google-github-actions/auth@v2
        with:
          workload_identity_provider: ${{ secrets.WIF_PROVIDER }}
          service_account: ${{ secrets.WIF_SERVICE_ACCOUNT }}

      - name: Deploy to Cloud Run
        uses: google-github-actions/deploy-cloudrun@v2
        with:
          service: ${{ secrets.CLOUD_RUN_SERVICE }}
          region: ${{ secrets.GCP_REGION }}
          image: ${{ needs.build-and-push.outputs.full-image }}
          env_vars: |
            SPRING_PROFILES_ACTIVE=prod
            DB_URL=${{ secrets.DB_URL }}
            DB_USERNAME=${{ secrets.DB_USERNAME }}
            DB_PASSWORD=${{ secrets.DB_PASSWORD }}
            JWT_SECRET=${{ secrets.JWT_SECRET }}
          flags: |
            --allow-unauthenticated
            --port=8080
            --memory=512Mi
            --cpu=1
            --min-instances=0
            --max-instances=3

      - name: Print service URL
        run: echo "Deployed to ${{ steps.deploy.outputs.url }}"
```

## Common failures & fixes

| Symptom | Cause | Fix |
|---|---|---|
| `Permission denied on Artifact Registry` | SA missing `artifactregistry.writer` | Re-run IAM binding from Pre-requisites C |
| `Unable to generate OIDC token` | WIF provider attribute condition mismatch | Check `GITHUB_ORG/GITHUB_REPO` values in provider |
| `Container failed to start` | Missing env var in Cloud Run | Add missing var to `env_vars` block in workflow |
| `Flyway migration error` | Schema mismatch vs prod DB | Run `flyway repair` then redeploy |
| `Tests fail in CI but pass locally` | Active profile not set | Confirm `-Dspring.profiles.active=h2` in test step |
